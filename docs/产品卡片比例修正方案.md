# 产品卡片比例修正方案

> 日期：2026-06-14
> 状态：步骤1 ✅ / 步骤2 ✅ / 步骤3 ✅ / 步骤4 待定

## 问题

产品中心页面卡片（`.product-card`）有两个问题：
1. 图片区占比偏低（42%），产品图展示不够突出
2. 卡片图有大量透明留白，电机主体实际只占画面 30%，显得过小

## 参考标准

以产品 1030837 在旧分支上的效果为基准：

| 维度 | 数值 |
|------|------|
| 卡片尺寸 | 654 × 654 px（1:1 正方形） |
| 图片区 | 654 × 470 px |
| 文字区 | 654 × 184 px |
| **图片占比** | **470 / 654 ≈ 71.9%** |

## 步骤 2：修复 Canvas 上传破坏图片 ✅ 已完成

**文件**：`backend/admin.html` — `handleCardImgSelect()`

### 移除的操作

| 操作 | 影响 |
|------|------|
| `fillStyle='#ffffff'` + `fillRect` 填充白底 | 破坏 PNG 透明通道，透明区变不透明白 |
| 15% 四周留白 + canvas 放大 | 电机在画面中占比缩小 |

|------|------|----------|------|

| 损坏版 | `1061418_78785272.png` | `(255,255,255,255)` 不透明白 | 旧 Canvas fillRect 破坏 |
| 修复后 | `1061418_f5e0f941.png` | `(255,255,255,255)` 仍白 | 用户反复上传已损坏的源图 |

## 步骤 3：Canvas 自动裁边 + 正方形输出 ✅ 已完成

**文件**：`backend/admin.html` — `handleCardImgSelect()`

上传卡片图时的处理流程：

1. **绘制到临时 canvas** → 获取像素数据
2. **采样扫描非透明像素** → 找到内容边界框（stride=min(w,h)/400，性能安全）
3. **计算正方形裁剪区** → 以内容中心为锚点，取 max(contentW, contentH) 为边长，加 10% 边距
4. **输出正方形透明底 PNG** → `drawImage` 从原图裁剪目标区域

### 效果对比（1061418 实测）

| 阶段 | 图片尺寸 | 电机高度占比 | 说明 |
|------|----------|-------------|------|
| 原始 | 3992×3992 | 30.1% | 大量透明留白 |
| 自动裁边后 | 3126×3126 | 38.5% | 裁掉留白，电机更大 |

### 1061418 当前 cardImage

已用 Python 脚本等效处理：`images/products/1061418_8e4f281c.png`（3126×3126，透明底）

### 对其他产品的影响

后续通过管理后台上传卡片图时，Canvas 会自动裁边 + 正方形输出。其他产品的卡片图如果也是透明底+大量留白，上传后会自动获得相同的裁边效果，视觉比例趋近一致。

## 步骤 4：`object-fit` 策略（待定）

当前 `build.py` 仍用 `object-fit: cover`。裁边后图片已为正方形，`cover` 在正方形图片上等同于 `contain`/`fill`（无裁切）。暂不调整。
